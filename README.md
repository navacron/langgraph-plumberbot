# langgraph-plumberbot

A series of progressively more complex LangGraph examples built around a plumbing service triage scenario.

| Folder | Description |
|--------|-------------|
| [`langgraph-plumberbot-bare-01`](./langgraph-plumberbot-bare-01) | **Starter MVP** — deterministic keyword logic, no LLM calls; demonstrates StateGraph, typed state, conditional routing, `interrupt()`, and checkpointed human-in-the-loop approval. |
| [`langgraph-plumberbot-withllm-02`](./langgraph-plumberbot-withllm-02) | **LLM-powered** — same graph topology, but classification uses Claude with structured JSON output + adaptive thinking, and every response is generated by the Anthropic API (`anthropic` SDK). |
| [`langgraph-plumberbot-langsmith-03`](./langgraph-plumberbot-langsmith-03) | **Cloud deployment** — same graph deployed to LangGraph Cloud; CLI uses `langgraph-sdk` to invoke over HTTP, detect interrupts via thread state, and resume — the same pattern a Next.js UI would use. |
| [`langgraph-plumberbot-hugging-04`](./langgraph-plumberbot-hugging-04) | **HF Spaces web demo** — Gradio UI + SQLite checkpointer (`SqliteSaver`) deployed as a single Docker container on Hugging Face Spaces; no LangGraph Server required. |
